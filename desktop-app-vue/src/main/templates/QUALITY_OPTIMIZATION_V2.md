## 🎨 模板1：活动海报设计方案

### 优化前状态（87分）

**存在问题**：
- ❌ 变量太少（7个），灵活性不足
- ❌ Prompt过于简略，缺少具体RGB值、字体大小等细节

- ❌ 文件输出单一（仅2个）

**新增变量列表**：
1. `colorScheme` - 色彩方案（暖色系/冷色系/对比色/单色系）
2. `urgencyLevel` - 紧迫感等级（高/中/低）
3. `ctaText` - 行动号召文案（立即参与/马上报名/了解详情等）
4. `hasQRCode` - 是否包含二维码
5. `includeSponsors` - 是否包含赞助商Logo

#### ✅ Prompt大幅增强（2倍内容）

**新增内容模块**：

1. **海报尺寸与规格**（4种尺寸详细说明）
   - 手机竖屏(9:16)：1080×1920px，安全区域详细标注
   - 方形(1:1)：1080×1080px
   - 横屏(16:9)：1920×1080px
   - A4打印：2480×3508px，含出血位

2. **色彩方案详细规范**（4套完整配色）
   ```
   暖色系方案：
   - 主色：#FF6B35 RGB(255, 107, 53)
   - 辅色1：#FFD23F RGB(255, 210, 63)
   - 辅色2：#EE4E34 RGB(238, 78, 52)
   - 背景色：#FFF8F0 RGB(255, 248, 240)

   + 冷色系、对比色、单色系各一套
   ```

3. **文案策划**（根据活动类型动态生成）
   - 促销活动：限时折扣文案
   - 发布会：震撼标题建议
   - 线上活动：扫码引导语
   - 线下活动：地点突出

4. **版面布局方案**（5种设计风格）
   - 现代简约：留白30%，几何图形装饰
   - 科技感：渐变背景，发光效果
   - 国潮风：传统元素，中轴对称
   - 复古风：纹理背景，装饰边框
   - 扁平化：纯色块，图标化

5. **详细设计规范**
   - 字号规范：根据海报尺寸动态调整（主标题72-120pt）
   - 间距规范：标题间距30-40px，元素边距≥20px
   - 按钮设计：圆角8-12px，阴影0 4px 12px

6. **设计心理学应用**
   - 对比度要求：文字与背景≥4.5:1
   - F型视觉流：重点信息布局
   - 色彩心理：红橙=紧迫、蓝绿=专业

**新增文件**：
1. `设计规范.md` - 快速参考的规范表
2. `素材需求清单.md` - 图片/图标需求checklist

#### ✅ Handlebars高级用法

## 🎯 优化技术总结

### 1. 变量设计策略

**分层变量设计**：
- **基础变量**：名称、类型、时长等（必填）
- **差异化变量**：级别、规模、阶段等（动态调整内容）
- **开关变量**：是否包含XX（模块化控制）

**变量类型使用**：
- `string`：自由文本输入（34%）
- `number`：数值范围（21%）
- `select`：单选下拉（45%，最常用）

### 2. Prompt优化技巧

**结构化Prompt**：
1. 使用Markdown标题分级（##、###）
2. 表格展示规范数据（色值RGB、字号、时间分配）
3. 列表展示操作步骤
4. 代码块展示配置示例

**动态内容生成**：
- 根据变量值提供4-5种不同方案
- 使用`{{#if}}`和`{{else if}}`实现多分支
- 嵌套条件：最多3层

**细节量化**：
- 色值：提供HEX和RGB双格式
- 尺寸：提供px和pt双单位
- 比例：提供百分比（留白30%）

### 3. 文件结构设计

**多文件输出原则**：
1. **主文档**（Word/PPT）：核心内容
2. **辅助文档**（Markdown）：清单、手册、指导
3. **数据文件**（Excel）：表格、进度跟踪

**Markdown文件优势**：
- 支持Handlebars变量插值
- 支持条件渲染
- 轻量级，易于版本控制
- 可导出PDF分享

### 4. Handlebars高级用法

**条件判断**：
```handlebars
{{#if (eq variable "value")}}
  内容1
{{else if (eq variable "value2")}}
  内容2
{{else}}
  默认内容
{{/if}}
```

**循环生成**：
```handlebars
{{#each (range 1 days)}}
  第{{this}}天：...
{{/each}}
```

**默认值**：
```handlebars
{{default optional_variable "默认值"}}
```

## 💡 最佳实践建议

### 对于模板设计者

1. **变量数量**：建议8-12个，不宜超过15个
2. **Prompt长度**：主文档800-1200行，辅助文档200-400行
3. **条件渲染**：至少5处，覆盖主要差异化场景
4. **文件输出**：至少3个（1个主文档+2个辅助）

### 对于用户

1. **快速上手**：必填变量≤3个，其他使用默认值
2. **深度定制**：逐步调整可选变量，观察输出变化
3. **保存方案**：将常用变量组合保存为预设

### 对于开发者

1. **JSON验证**：确保所有双引号正确转义
2. **Handlebars测试**：测试所有条件分支
3. **文件路径**：使用变量插值生成动态文件名
