## 1. 文件压缩工具

### 压缩文件 📦

**使用场景**：需要压缩文件或文件夹，节省存储空间或便于传输

**操作步骤**：
```
1. 选择要压缩的文件
2. 设置输出路径和压缩格式
3. 可选：设置压缩级别和密码
4. 执行压缩
```

**示例**：
```javascript
压缩项目文件夹:
- 文件: ['项目文档/', '图片/', 'README.md']
- 输出: 'project-backup.zip'
- 格式: ZIP
- 压缩级别: 最大压缩
- 密码: my_secure_pass
```

**支持格式**：ZIP、RAR、7Z、TAR.GZ

**压缩级别说明**：
- 仅存储 - 不压缩，速度最快
- 最快 - 压缩率低，速度快
- 快速 - 平衡速度和压缩率
- 标准 - 推荐设置
- 最大 - 压缩率高，速度慢
- 超级 - 压缩率最高，速度最慢

### 解压文件 📂

**使用场景**：解压ZIP、RAR等压缩文件

**操作步骤**：
```
1. 选择压缩包
2. 设置解压目录
3. 如果有密码，输入密码
4. 执行解压
```

**提示**：可以选择性解压特定文件

## 10. 网络诊断工具

### 网速测试 📊

**使用场景**：测试网络速度

**操作步骤**：
```
1. 选择测试类型：
   - 仅下载
   - 仅上传
   - 同时测试
   - 仅Ping
2. 可选：选择测试服务器
3. 开始测试
4. 查看结果
```

**结果解读**：
```
下载速度：125.45 Mbps
上传速度：38.21 Mbps
延迟(Ping)：15.23 ms
抖动：2.15 ms
质量评级：优秀
```

**质量评级**：
- 优秀：延迟 < 20ms
- 良好：延迟 20-50ms
- 一般：延迟 50-100ms
- 较差：延迟 > 100ms

### 网络诊断 🔧

**可用诊断**：
1. **Ping测试** - 测试连通性和延迟
2. **端口扫描** - 检查端口开放状态
3. **DNS查询** - 查询域名解析
4. **路由追踪** - 追踪网络路径
5. **WHOIS查询** - 查询域名信息

**Ping测试示例**：
```
测试Google连通性:
- 目标：www.google.com
- 次数：4次
- 超时：1000ms

结果：
- 平均延迟：25.6ms
- 丢包率：0%
```

**端口扫描示例**：
```
检查Web服务:
- 目标：example.com
- 端口：80, 443, 22, 3306

结果：
- 80 (HTTP)：开放
- 443 (HTTPS)：开放
- 22 (SSH)：关闭
- 3306 (MySQL)：关闭
```

**DNS查询示例**：
```
查询域名解析:
- 域名：example.com
- DNS服务器：8.8.8.8

结果：
- A记录：192.0.2.1
- MX记录：mail.example.com
- NS记录：ns1.example.com
```

## ❓ 常见问题

### Q1: 压缩文件后为什么还是很大？
**A**: 某些文件（如已压缩的视频、图片）再次压缩效果不明显。建议：
- 对文档、代码等文本文件压缩效果更好
- 尝试使用"最大"或"超级"压缩级别
- 考虑使用7Z格式，压缩率更高

### Q2: 视频编辑后质量下降？
**A**: 建议：
- 使用H.264或H.265编码器
- 提高码率设置（5-10 Mbps）
- 使用"高"或"超高"质量设置

### Q3: 二维码扫描不出来？
**A**: 可能原因：
- 图片模糊或分辨率太低
- 二维码损坏或遮挡
- 尝试增加容错级别后重新生成

### Q4: 忘记密码库主密码怎么办？
**A**:
⚠️ 主密码无法找回！建议：
- 设置易记但难猜的主密码
- 将主密码安全保存（如写在纸上）
- 定期导出密码库备份

### Q5: 笔记可以导出吗？
**A**: 可以：
- Markdown格式可直接复制
- 使用文件压缩器打包导出
- 后续版本将支持批量导出
