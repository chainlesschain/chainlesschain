# but are NOT tracked in git. They live in a dedicated GitHub Release
# `internal-binaries-android-v<binariesVersion>`. binariesVersion is set in
# android-app/gradle.properties.
#
# Format: <sha256>  <release-asset-name>  <target-path-relative-to-repo-root>
#
# `downloadBootstrapBinaries` Gradle task (defined in
# feature-local-terminal/build.gradle.kts) + `downloadFridaBinaries`
# (defined in app/build.gradle.kts) read this manifest, ensure each
# target exists with matching sha256, downloading from
# https://github.com/chainlesschain/chainlesschain/releases/download/internal-binaries-android-v$BINARIES_VERSION/<asset-name>
# if missing or stale. Verified files become part of the APK at packageRelease time.
#
# To refresh: the node-runtime-bundle.yml workflow rebuilds, uploads to a
# new release tag, then bumps binariesVersion in gradle.properties via a
# tiny text-only commit (replaces the prior auto-commit-binaries pattern
