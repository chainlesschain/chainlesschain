## 摘要

ChainlessChain（无链之链）是厦门无链之链科技有限公司自主研发的一体化人工智能办公系统，面向党政机关、事业单位和大中型国企的"信创+AI+协同办公"综合需求设计。

**核心特征：**

1. **全栈国产化** — 从 CPU、操作系统、数据库到大模型层全部支持国产替代方案
2. **数据完全本地化** — 所有数据存储在用户单位自有设备和服务器上，永不上传第三方
3. **硬件密钥保护** — 采用 U盾、SIM 安全密钥、移动终端硬件安全模块作为身份根
4. **架构原生符合等保三级** — 不需要二次改造即可通过等级保护测评
5. **AI 能力本地可运行** — 整套系统在断网状态下仍可正常使用
6. **一次买断永续使用** — 不存在订阅锁定、不依赖境外服务可用性

**适用场景：**

党政机关公文起草与审批协同、事业单位知识管理、国企内部协作 IM、跨单位涉密通信、行业智能助手部署、信创替代项目。

**技术成熟度：**

产品已发布稳定版本，全部功能模块已通过自动化测试验证（累计测试用例超过 15,000 条），具备生产环境部署能力。

## 第一章 行业背景与战略契合

### 1.1 国家信创战略

党的二十大报告明确提出"加快建设网络强国、数字中国"，强调"健全关键信息基础设施安全保护体系"。十四五规划明确提出推进信息技术应用创新（信创），实现关键软硬件自主可控。

主要政策依据：

| 政策法规 | 核心要求 |
|---|---|
| 《数据安全法》（2021 年 9 月施行）| 重要数据不得擅自向境外提供 |
| 《个人信息保护法》（2021 年 11 月施行）| 个人信息处理需明确告知用途、最小必要原则 |
| 《网络安全法》（2017 年 6 月施行）| 关键信息基础设施运营者落实等级保护 |
| 《关键信息基础设施安全保护条例》（2021 年 9 月） | 关基采购的网络产品和服务可能影响国家安全的需通过审查 |
| 《密码法》（2020 年 1 月施行）| 涉密领域必须使用商用密码或国密算法 |
| 信息系统安全等级保护 2.0 标准 | 党政关键系统需达到等保三级或以上 |

### 1.2 党政机关数字化办公的现实挑战

#### 1.2.1 AI 工具使用失控

国内大量党政机关、事业单位员工在工作中私下使用 ChatGPT、Claude、Gemini 等境外 AI 工具，将本应保密的公文草稿、会议纪要、研究材料粘贴给境外服务进行处理。这一行为已构成：

- 违反《数据安全法》关于重要数据不出境的规定
- 违反保密法律法规
- 形成不可控的数据外泄通道

通过单纯的"行政禁令"难以根治，必须**以国产合规替代品填补需求空缺**。

#### 1.2.2 进口办公软件的安全隐患

Microsoft 365、Google Workspace 等境外办公套件存在以下问题：

- **数据存放境外**：默认数据中心在美国 / 欧洲，跨境传输受限
- **服务可被切断**：政治、贸易因素可能导致服务中断（已有先例）
- **暗箱遥测**：客户端遥测内容、范围、频率均不透明
- **不符合等保三级**：身份认证、数据加密、审计日志机制与国标存在差异

#### 1.2.3 互联网商业 IM 的合规盲区

钉钉、企业微信、飞书等被广泛用于工作沟通，但存在：

- **数据托管在商业公司服务器**：不符合涉密要求
- **缺乏端到端加密**：服务商技术上可读取所有内容
- **审计能力受限**：单位无法对本单位人员的通信进行完整审计
- **超出工作场景的数据沉淀**：员工的工作内容沉淀在商业平台，离职后丢失

#### 1.2.4 系统割裂、重复采购

党政机关单位往往同时采购：OA 系统、邮箱系统、文件服务器、即时通信、知识库、AI 助手等多套系统，存在：

- 数据无法互通，员工需在多个系统间切换
- 重复采购，TCO 高
- 多套账号、多重密码，安全管理困难
- 各系统供应商不同，统一维保困难

### 1.3 信创办公替代的市场窗口

党政机关、央企、国资企业的信创替代正全面推进。当前市场上的同类产品大多停留在"国产 OA + 国产 IM"的层面，**真正"AI + 知识库 + IM + 数字资产"一体化、全栈国产化的产品稀缺**。这正是 ChainlessChain 的市场窗口。

### 3.2 三层架构

ChainlessChain 系统分为三个逻辑层：

#### 3.2.1 客户端层

提供给用户的具体操作界面：

- **桌面应用**：基于 Electron + Vue 3 技术构建，覆盖 Windows、macOS、Linux 三大桌面平台
- **移动应用**：Android 端基于 Jetpack Compose 原生开发；iOS 端基于 SwiftUI 原生开发
- **命令行工具**：基于 Node.js，跨平台单一可执行文件，无需图形界面即可使用

#### 3.2.2 数据与计算层

数据与计算全部在客户单位本地完成：

- **加密数据库**：所有结构化数据存放在 AES-256-GCM 加密的 SQLite 中
- **向量索引**：用于语义检索的向量数据库（开源 Qdrant，可替换为国产产品）
- **本地大模型**：通过 Ollama 等推理引擎运行国产大模型（通义千问 / 文心 / 星火 / DeepSeek 等）
- **文件存储**：用户文件存储在本地或单位文件服务器，路径可配置

#### 3.2.3 通信与同步层

各客户端之间的数据同步与通信：

- **本地局域网直连**：同一局域网内的设备直接通信，无需经过任何外部服务器
- **加密 P2P 网络**：跨地域时通过加密 P2P 通信
- **可选信令服务器**：用于建立 P2P 连接的辅助服务器，可由客户单位自行部署
- **可选中继服务器**：在严格 NAT 网络下作为中转，可由客户单位自行部署

### 3.3 三端协同模式

| 端 | 主要角色 | 典型操作 |
|---|---|---|
| **桌面端** | 主要工作站 | 编写公文、AI 对话、文档管理、深度协作 |
| **移动端** | 现场延伸 | 拍照入库、语音录入、远程审批、外勤签名 |
| **命令行** | 后台 / 服务器 | 自动化任务、脚本集成、批量处理、CI/CD |

三端通过统一的硬件身份签名保证数据来源真实，所有操作的数据修改通过加密同步通道传播到其他端。

### 3.4 部署架构选项

根据客户单位规模和安全要求，提供四种部署架构：

#### 3.4.1 单机部署

每位用户在自己电脑上独立部署，数据各自保存。适合：
- 个人办公或小型办公室（< 20 人）
- 完全去中心化的轻量场景

#### 3.4.2 局域网部署

单位内部署一台服务器作为协同中心，员工设备通过局域网连接。适合：
- 中型单位（20–500 人）
- 内网办公环境

#### 3.4.3 私有云部署

部署在客户单位的私有云上，包含负载均衡、高可用、灾备等企业级特性。适合：
- 大型单位 / 集团（500+ 人）
- 多分支机构跨地域协同

#### 3.4.4 涉密内网部署

完全隔离的物理涉密内网中部署，配套国密算法、双因素认证、行为审计。适合：
- 党政涉密办公场景
- 按保密委要求定制实施

### 3.5 数据流向

ChainlessChain 严格遵循"数据不出域"原则。下表说明各类操作的数据流向：

| 操作 | 数据流向 |
|---|---|
| 用户编写文档 | 仅本地存储 |
| AI 智能问答（本地模型） | 全程本地，不出网 |
| AI 智能问答（远端模型） | 仅在用户明确选择时，向用户授权的远端 API 发送（默认关闭） |
| 知识库检索 | 全程本地 |
| 与同事 IM | 端到端加密，密文经局域网或加密 P2P 传输 |
| 跨单位通信 | 端到端加密，密文经加密 P2P 网络 |
| 多端同步 | 端到端加密，密文同步 |
| 系统遥测 | **无任何遥测**（默认关闭，不可向厂商回传）|

## 第四章 安全合规体系

### 4.1 等级保护 2.0 三级符合性

ChainlessChain 在系统架构设计阶段即按等保三级要求构建。下表对应等保三级关键控制项：

| 控制项类别 | 等保三级要求 | 我方实现 |
|---|---|---|
| **身份鉴别** | 双因素及以上 | 硬件密钥（U盾 / 密钥卡）+ 生物识别 / PIN |
| **访问控制** | 基于角色的最小权限 | 内置 RBAC 模型 + 项目级权限隔离 |
| **安全审计** | 全操作日志，不可篡改 | 加密签名审计日志 + 跨机分发归档 |
| **入侵防范** | 入侵检测与防御 | 异常行为检测 + 告警 + 应急响应 |
| **恶意代码防范** | 应有恶意代码防范机制 | 代码签名验证 + 沙箱执行 |
| **数据完整性** | 传输与存储完整性校验 | 端到端密码学校验 |
| **数据保密性** | 重要数据加密存储与传输 | AES-256-GCM 全程加密 |
| **数据备份恢复** | 关键数据定期备份与可恢复 | 自动备份 + 一键恢复 |
| **个人信息保护** | 最小必要 + 用户授权 | 完全本地存储，无任何上传 |
| **集中管控** | 安全管理中心统一管控 | 提供管理员控制台 |

### 4.2 《数据安全法》符合性

| 法律要求 | 我方实现 |
|---|---|
| 重要数据不得擅自出境 | 全部数据本地存储，无任何境外传输 |
| 关键信息基础设施安全保护 | 架构按关基保护要求设计，可通过国家网信办审查 |
| 数据分级分类管理 | 提供数据敏感度标签 + 访问策略 |
| 数据安全风险评估 | 提供完整的风险评估文档与工具 |

### 4.3 《个人信息保护法》符合性

| 法律要求 | 我方实现 |
|---|---|
| 知情同意 | 所有数据处理操作明确告知用户 |
| 最小必要 | 仅采集必要信息，无附加遥测 |
| 安全保护 | 全程加密 + 硬件密钥保护 |
| 用户权利 | 提供数据查阅、导出、删除完整能力 |
| 数据本地化 | 完全本地存储 |

### 4.4 《密码法》符合性

| 要求 | 我方实现 |
|---|---|
| 商用密码使用 | 默认采用国际通用密码（Ed25519 / AES-256-GCM / ML-KEM 等）|

### 4.6 密码体系详解

#### 4.6.1 已部署算法

| 用途 | 算法 |
|---|---|
| 数字签名 | Ed25519（高性能）+ ML-DSA / SLH-DSA（后量子）|
| 密钥协商 | X25519 + ML-KEM（后量子）|
| 对称加密 | AES-256-GCM |
| 哈希 | SHA-256 / SHA-3 |
| 即时通信加密 | Signal Protocol 双棘轮 |
| 数据库加密 | SQLCipher（AES-256）|

#### 4.6.2 国密算法适配规划

针对涉密领域客户，提供国密改造版本：

| 国密算法 | 用途 | 适配方式 |
|---|---|---|
| SM2 | 数字签名、密钥协商 | 替换默认 Ed25519/X25519 |
| SM3 | 哈希 | 替换默认 SHA-256 |
| SM4 | 对称加密 | 替换默认 AES-256-GCM |
| SM9 | 标识密码 | 可选集成 |

国密改造版本可申请商用密码产品认证。

#### 4.6.3 后量子密码（PQC）

应对未来量子计算机对传统密码的威胁，已内置 NIST 后量子密码标准：

- **ML-KEM（FIPS 203）**：抗量子密钥封装
- **ML-DSA（FIPS 204）**：抗量子数字签名
- **SLH-DSA（FIPS 205）**：基于哈希的抗量子签名

与传统算法形成混合签名，今天可用、明天可抗量子。

### 4.7 硬件信任根

ChainlessChain 采用三脚硬件信任根：

| 信任根 | 介质 | 解锁方式 |
|---|---|---|
| **U盾** | USB 硬件密钥（桌面端） | 物理插入 + PIN |
| **SIMKey** | SIM 卡安全域（手机端，规划中） | 双因素 |
| **移动硬件安全模块** | 手机芯片层安全飞地 | 生物识别 / PIN |

任何一脚都可独立签名；多脚同时启用时获得更高保护等级（如大额跨链交易、密钥轮换）。

### 4.8 审计与合规追溯

所有关键操作产生不可篡改的审计日志：

- 用户登录 / 登出
- 文件创建 / 修改 / 删除
- 多签提案 / 签名 / 执行
- 跨单位通信
- 系统配置变更
- 管理员操作

审计日志通过密码学签名 + Merkle 树存证，任何第三方可验证日志完整性。可定期归档到客户单位指定的存储介质（如蓝光归档库、防篡改存储）。

## 第五章 功能详解

### 5.1 智能办公助手

#### 5.1.1 AI 智能问答

用户可向 AI 提问任何问题，AI 既可基于内置知识回答，也可结合本单位文件库回答（智能检索增强）。

**典型用法：**

- "上个月我们部门的会议纪要里，关于 X 项目的进展是怎么说的？"
- "把这份合同与上次的合同对比，列出所有差异条款。"
- "起草一份给领导的关于 Y 议题的汇报材料，2000 字以内。"

回答会明确列出引用的具体文档段落，可点击跳转到原文核对。

#### 5.1.2 文档自动起草

支持以下文档类型自动起草：

- 公文：通知、报告、请示、批复、函等
- 报告：工作总结、专题报告、调研报告
- 演讲稿、发言稿
- PPT 大纲与各页要点
- Excel 数据分析报表
- 邮件回复
- 项目计划

起草时可指定：篇幅、语言风格（庄重 / 简洁 / 详尽）、引用的资料范围。

#### 5.1.3 会议纪要自动生成

录音 → 自动语音转文字 → 自动整理为标准纪要格式（议题、与会人员、主要观点、决议事项、待办事项）。

支持：

- 实时录音 + 实时转写
- 上传已有录音文件 + 批量转写
- 多人会议自动区分说话人
- 关键决议自动加粗标记

#### 5.1.4 跨文档对比与摘要

将多份文档（如多年同议题报告、多版本合同、多份调研材料）一次性提交给 AI，自动：

- 提取每份的核心观点
- 找出共同点与差异点
- 生成专题对比表
- 生成综合摘要

#### 5.1.5 内置技能库（139 项）

系统内置 139 项即装即用的 AI 技能，覆盖：

- **文档处理**：文档转换、PDF 处理、表格分析、PPT 制作
- **多媒体**：音频转写、视频处理、字幕生成、TTS 合成
- **数据**：CSV 处理、数据分析、图表生成
- **图像**：OCR 识别、图像生成、图像编辑
- **办公**：日程管理、任务跟踪、邮件起草
- **专业**：合规检查、法律条款分析、医疗信息处理
- **开发**：代码审查（仅限技术单位）

新技能可通过技能市场动态添加。

#### 5.1.6 多角色 AI 协同

复杂任务（如撰写完整调研报告、起草大型文件、深度分析项目）可通过"多角色 AI 协同"模式自动完成：

1. **理解** — AI 拆解需求
2. **规划** — AI 制定步骤
3. **执行** — 多个 AI 角色并行做事
4. **验证** — AI 自检并必要时迭代
5. **总结** — 输出最终结果与决策日志

每一阶段用户可中断、调整、手工接管。

### 5.2 知识库与协同办公

#### 5.2.1 统一知识库

把单位的所有信息资产汇集到一个加密知识库：

- 文档（Word / PDF / Excel / PPT / Markdown 等）
- 笔记（用户随时记录）
- 会议纪要
- 邮件归档
- 聊天记录
- 项目资料

每条资料带有：

- 创建人 / 创建时间 / 修改历史
- 标签 / 分类 / 项目归属
- 访问权限设置

#### 5.2.2 强大检索

提供两种检索方式：

- **关键词检索**：精确匹配字符串
- **语义检索**：按意思匹配（可找到同义不同字的内容）

两种检索结果合并加权，准确率与召回率双优。50 万条记录的库实测检索响应 < 200 毫秒。

支持高级检索语法：

- 按标签：`tag:工作`
- 按时间：`created:>2026-04-01`
- 按类型：`type:document`
- 短语精确：`"完整短语"`
- 排除词：`-排除词`

#### 5.2.3 项目空间

按项目 / 部门 / 主题划分独立空间，每个空间：

- 独立成员名单与权限
- 独立的资料、对话、AI 助手记忆
- 可选择性开放给外部成员协作
- 项目结束可整体归档

#### 5.2.4 多设备同步

办公室电脑、笔记本、家庭电脑、手机数据自动同步：

- 局域网内直接同步（速度最快）
- 外出时通过加密 P2P 网络
- 全程不经过任何第三方云
- 支持选择性同步（如只同步 A 项目）

#### 5.2.5 版本与审计

每份文档的修改历史完整保留：

- 谁在何时改了什么
- 可一键回滚到任意历史版本
- 关键文档可锁定历史版本

### 5.3 加密即时通信

#### 5.3.1 一对一加密对话

任意两位用户之间的对话端到端加密。即使消息经过中转服务器，服务器也只能看到密文。

支持：

- 文字
- 图片 / 文件
- 引用回复
- 表情
- 已读回执（可关闭）

#### 5.3.2 多人加密群组

群组对话同样端到端加密。支持：

- 自由创建群组
- 群主 / 管理员 / 成员三级权限
- 公告 / 文件柜 / 投票
- 历史消息搜索

#### 5.3.3 频道讨论

按主题划分的频道，类似 Slack channel：

- 公开频道：单位内任何人可加入
- 私有频道：邀请制
- 公告频道：仅管理员可发言
- 频道内消息按主题归档

#### 5.3.4 公文流转

替代传统的纸质 / 邮件公文流转：

- 起草人发起 → 部门主管审核 → 单位领导签发
- 全程加密留痕
- 多签自动收集
- 完成后自动归档到知识库

#### 5.3.5 多签决策

针对"三重一大"（重大决策、重大项目、重要人事任免、大额资金使用）等场景：

- 发起人提出动议
- 系统自动按规则路由到相关签批人
- 每位签批人独立硬件密钥签名
- 达到指定数量的签名后自动生效
- 全过程加密留痕

#### 5.3.6 跨单位互通

部署了本系统的单位之间可建立加密通信通道：

- 双方各派代表交换 DID（去中心化身份证）
- 之后该单位的所有成员都可与对方单位授权成员通信
- 通信全程端到端加密
- 不依赖任何第三方平台

### 5.4 数字资产管理（可选模块）

针对需要管理数字资产的单位（如金融、央企、跨境业务）：

- 多链钱包管理
- 跨链原子交换
- 大额多签强制
- 完整审计追溯

普通党政办公场景可不启用此模块。

### 5.5 远程操控

针对"领导出差，需要在手机上指挥办公室电脑做事"的场景：

- 手机 ↔ 电脑安全配对（扫码 1 秒完成）
- 手机端可远程查看电脑屏幕状态
- 手机端可远程发起 AI 任务、文件检索、签名审批
- 手机端可远程打开电脑终端（命令行），执行任意操作
- 高危操作（删除文件 / 系统命令）触发电脑端二次确认

通信全程端到端加密，全程可审计。

## 第十章 典型应用场景

### 10.1 党政机关办公场景

**场景：** 某省级党政机关需要替代既有的国外办公套件，建立完全自主可控的智能化办公环境。

**部署方案：**

- 私有云部署，包含主备双中心
- 1500 用户授权，覆盖全机关
- 接入既有 OA 与邮件系统
- 配置内部加密 IM 替代微信工作群
- 部署本地大模型（70B）用于公文起草与材料检索
- 集成国密算法

**典型工作流：**

1. 处级干部撰写请示稿，AI 助手协助打磨措辞
2. 通过加密 IM 发起部门内部讨论
3. 形成定稿后通过多签流程报领导签批
4. 签批后自动归档到机关知识库
5. 后续相关项目检索可一键调出此请示及背景材料

**关键收益：**

- 数据完全本地，符合保密要求
- 工作效率显著提升（公文起草时间缩短）
- 知识资产沉淀，避免人员调动造成断层

### 10.2 央企集团总部场景

**场景：** 某央企集团总部 + 全国 30 个分子公司，需建立统一的协同与知识管理平台。

**部署方案：**

- 总部主集群 + 各分子公司分布式节点
- 5000 用户授权
- 跨地域加密 P2P 通信
- 总部统一管理 + 各公司本地数据自治
- 接入集团 ERP / 财务系统

**典型工作流：**

1. 总部下达指令通过加密 IM 推送给各分子公司
2. 分子公司用本地 AI 起草响应材料
3. 重大投资决策走总部 - 分公司 - 业务部门多签流程
4. 每月各分子公司经营数据自动汇总到总部 BI 看板
5. 集团知识库统一沉淀战略、政策、最佳实践

**关键收益：**

- 避免使用境外协同工具的合规风险
- 各分子公司业务数据保持本地，符合属地监管
- 跨地域协同效率提升

### 10.3 涉密科研单位场景

**场景：** 某国防科研单位需要在涉密内网中部署 AI 助手，但严禁与外网任何连接。

**部署方案：**

- 涉密内网完全隔离部署
- 全套国密算法改造版本
- 本地大模型（无远端 API）
- 软件更新通过涉密专网或物理介质
- 100 用户授权
- 双因素认证 + 行为审计

**典型工作流：**

1. 研究员用 AI 助手检索本单位历史涉密论文与项目资料
2. AI 协助起草技术报告、专利申请
3. 项目组内部加密讨论
4. 涉密文档全程不离开内网
5. 完整操作日志留痕，可应对国家保密检查

**关键收益：**

- 在涉密环境引入 AI 能力，提升研发效率
- 完全符合涉密信息系统管理要求
- 通过保密委审查的可能性高

### 10.4 大型律师事务所场景

**场景：** 某全国百强律师事务所，500+ 律师，需要客户案卷管理 + AI 辅助检索 + 客户保密通信一体化方案。

**部署方案：**

- 总所私有云部署
- 各分所通过加密通道连入
- 500 用户 + 200 助理用户
- 案卷加密存储
- 与客户之间建立加密通信通道

**典型工作流：**

1. 律师录入案卷资料（合同、证据、判决等）
2. AI 助手协助检索同类案件
3. AI 协助起草法律文书
4. 与客户通过加密 IM 沟通，符合律师保密义务
5. 多名律师协作时统一项目空间

**关键收益：**

- 客户保密义务履行更扎实（端到端加密）
- 律师工作效率提升
- 案卷资产沉淀于事务所
