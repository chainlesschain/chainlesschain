# 微信 / QQ / 抖音 / 头条 数据库表结构详解（字段含义 + 表关联）

> 解密后聊天/社交库的人话版字典。每张核心表逐字段中文备注 + 表之间怎么 JOIN。
> 自动 dump 版见 `SCHEMA_字典.md`；本文件是**人工详解版**，重点讲"每个字段啥意思、表怎么关联"。
> 解密脚本见 `scripts/android/pdh-*-decrypt.mjs`，runbook `docs/internal/pdh-wechat-decrypt-runbook.md`。

## 0. 跨 App 通用心智模型

四个 App 的聊天数据都是同一套「**消息 — 会话 — 联系人**」三元结构，只是字段名不同：

| 概念 | 微信 | QQ NT | 抖音/头条(同 ByteDance 框架) |
|---|---|---|---|
| 一条消息 | `message` 表一行 | `c2c_msg_table`/`group_msg_table` 一行 | `msg` 表一行 |
| 会话/群 | `chatroom`(群) + talker 字段 | `group_list`/`group_detail_info_ver1` | `conversation_core`/`conversation_list` |
| 联系人 | `rcontact` | `profile_info_v6`/`buddy_list` | `participant`(成员) |
| "谁发的" | message.talker 内嵌 `wxid:` | `40033`(QQ号) | `msg.sender` |
| "在哪个会话" | message.talker | `40021`(群号) | `msg.conversation_id` |
| 时间 | 毫秒 | **秒** | 毫秒 |

**关联主线**：消息.发送者 → 联系人表(取昵称)；消息.会话/群 → 群表(取群名)。下面每个 App 给出具体 JOIN 键。

## 1. 微信 EnMicroMsg.db（安卓，SQLCipher 解密后）

### 1.1 `message` — 聊天消息（核心表，52129 行）

| 字段 | 类型 | 含义 | 详细说明 |
|---|---|---|---|
| `msgId` | INTEGER PK | 本地消息ID | 本机自增，仅本地唯一 |
| `msgSvrId` | INTEGER | 服务器消息ID | 全局唯一，跨设备同一条消息此值相同；可用于去重 |
| `type` | INT | **消息类型** | `1`=文本 `3`=图片 `34`=语音 `42`=名片 `43`=视频 `44`=视频通话邀请 `47`=动画表情 `48`=位置 `49`=分享/链接/文件/转账/小程序(看 content 里 `<appmsg>` 的 `<type>` 子类) `50`=语音/视频通话 `10000`/`10002`=系统提示(撤回/拍一拍/入群) |
| `status` | INT | 发送状态 | `2`=发送中 `3`=已送达 `4`=已读等 |
| `isSend` | INT | **方向** | `0`=收到(对方发的) `1`=我发出的 |
| `createTime` | INTEGER | **时间戳(毫秒)** | `new Date(createTime)` 直接是本地时间 |
| `talker` | TEXT | **会话方(关联键)** | 单聊=对方 `wxid_xxx`；群聊=`<数字>@chatroom`；公众号=`gh_xxx`。**JOIN rcontact.username / chatroom.chatroomname 用它** |
| `content` | TEXT | **正文** | 文本消息=纯文本；**群消息会带前缀 `wxid_发送者:\n实际内容`**(用来标识群里谁发的)；type=49 时是 XML(`<appmsg>` 含标题/链接/文件名) |
| `imgPath` | TEXT | 缩略图/资源路径 | 图片/视频的本地缓存路径或 thumb key |
| `reserved` | TEXT | 保留字段 | 部分类型放扩展信息 |
| `lvbuffer` | BLOB | 扩展二进制 | 部分消息的 protobuf 扩展 |
| `transContent` | TEXT | 翻译内容 | 长按翻译的结果 |
| `talkerId` | INTEGER | 会话方数字ID | 关联 rcontact 的 rowid(部分版本) |
| `bizChatId` | INTEGER | 企业微信会话ID | 默认 -1 |
| `msgSeq` | INTEGER | 消息序号 | 会话内顺序 |

> **取群消息发送者**：`content` 按第一个 `:` 切，前半是 `wxid_发送者`，去 rcontact 查昵称；后半是真正内容。单聊没有此前缀。

### 1.2 `rcontact` — 联系人/好友/群/公众号（1926 行）

| 字段 | 类型 | 含义 | 说明 |
|---|---|---|---|
| `username` | TEXT PK | **账号(主键/关联键)** | `wxid_xxx`(人) / `<id>@chatroom`(群) / `gh_xxx`(公众号)。**= message.talker** |
| `alias` | TEXT | 微信号 | 用户自设的微信号(可搜索的那个) |
| `conRemark` | TEXT | **我给的备注名** | 优先显示这个；空则显示 nickname |
| `nickname` | TEXT | **对方昵称** | 对方自己设的名字 |
| `type` | INT | 联系人类型(位标志) | 含好友/群/公众号/黑名单等位 |
| `pyInitial` | TEXT | 昵称拼音首字母 | 用于字母索引/搜索 |
| `quanPin` | TEXT | 昵称全拼 | 同上 |

### 1.3 `chatroom` — 群信息（47 行）

| 字段 | 类型 | 含义 | 说明 |
|---|---|---|---|
| `chatroomname` | TEXT PK | **群ID** | `<数字>@chatroom`。**= message.talker(群聊时)** |
| `memberlist` | TEXT | **群成员 wxid 列表** | `;` 分隔的一串 wxid |
| `displayname` | TEXT | 群成员群昵称 | `;` 分隔，**与 memberlist 一一对应**(同序) |
| `roomowner` | TEXT | 群主 wxid | |
| `selfDisplayName` | TEXT | 我在本群的昵称 | |

### 1.4 微信关联关系

```
message.talker ──┬─(单聊/公众号)─→ rcontact.username  (取 conRemark/nickname 显示名)
                 └─(群,@chatroom)─→ chatroom.chatroomname ──→ memberlist/displayname(成员)
                                                          └→ roomowner ─→ rcontact.username(群主)
群消息发送者: message.content 前缀 "wxid_xxx:" ─→ rcontact.username
图片消息: message(type=3) ──→ ImgInfo2(7100行, 按 msgSvrId 关联图片大图/缩略图 key)
```

## 2. QQ NT（nt_msg.db + profile_info.db + group_info.db，同账号同密钥解密）

> QQ NT 用**数字列名**(40xxx/60xxx/1xxx)。时间是**秒**。消息正文是 protobuf。

### 2.1 `group_msg_table` / `c2c_msg_table` — 群聊 / 单聊消息

| 列 | 类型 | 含义 | 说明 |
|---|---|---|---|
| `40001` | INTEGER PK | **msgId** | 雪花算法 19 位长整型(超 JS 安全整数，读取必须当字符串/BigInt) |
| `40002` | INTEGER | 消息 random | 客户端随机数，去重用 |
| `40003` | INTEGER | 消息 seq | 会话内序列号 |
| `40020` | TEXT | **发送者 uid** | `u_xxx`(注意：不是消息ID，是发消息那个人的 uid)。JOIN profile_info_v6.`1000` |
| `40021` | TEXT | **群号 / 对端QQ号(关联键)** | 群聊=群号；单聊=对方QQ号。**JOIN group_info.`60001` 取群名** |
| `40027` | INTEGER | 对端 uid 数字 | |
| `40030` | INTEGER | 发送者类型 | |
| `40033` | INTEGER | **发送者 QQ号(uin)** | **JOIN profile_info_v6.`1002` 取昵称**(群里谁发的) |
| `40090` | TEXT | 发送者群名片/昵称 | 群里显示名(冗余存了一份) |
| `40093` | TEXT | 群名片/备注 | |
| `40010` | INTEGER | 消息归属 | `0`=自己 `2`=群 |
| `40011` | INTEGER | **消息类型(大类)** | `2`=普通图文消息 `5`/`9`=灰条系统消息(撤回/入群提示，分析时通常跳过) |
| `40012` | INTEGER | 消息子类型(elemType) | 富文本元素类型 |
| `40013` | INTEGER | 消息状态 | |
| `40050` | INTEGER | **时间戳(秒)** | ×1000 才是毫秒 |
| `40040` | INTEGER | 已读标志 | |
| `40800` | BLOB | **消息体(protobuf)** | 富文本元素(文本/图片/@/引用/文件)编码在此；纯文本可递归提取 UTF-8 字符串得到 |
| `40900` | BLOB | 消息扩展(protobuf) | |
| `40105` | INTEGER | 引用消息信息 | 回复哪条 |

### 2.2 `profile_info_v6` — 联系人资料（946 行）

| 列 | 类型 | 含义 | 说明 |
|---|---|---|---|
| `1000` | TEXT | **uid** | `u_xxx`。**= 消息 40020** |
| `1002` | INTEGER | **QQ号(uin)** | **= 消息 40033**，关联键 |
| `20002` | TEXT | **昵称** | 显示名 |
| `20009`/`20011` | | 性别/生日等 | |

### 2.3 `buddy_list` — 好友列表（781 行）

| 列 | 含义 | 说明 |
|---|---|---|
| `1000` | uid | = profile_info_v6.1000 |
| `1002` | QQ号 | = profile_info_v6.1002 |
| `25007` | 好友分组/标志 | |

> `profile_info_v6`(946) ⊇ `buddy_list`(781)：profile 含好友 + 陌生人/群友缓存；buddy_list 只是确认的好友。

### 2.4 `group_list` / `group_detail_info_ver1` — 群信息（各 108 行）

| 列 | 类型 | 含义 | 说明 |
|---|---|---|---|
| `60001` | INTEGER | **群号(关联键)** | **= 消息 40021**(实测 link 列) |
| `60004` | INTEGER | 群主 QQ号 | JOIN profile_info_v6.1002 |
| `60007` | TEXT | **群名** | 显示名 |
| `60005` | INTEGER | 最大成员数 | |
| `60006` | INTEGER | 当前成员数 | |
| `60002` | TEXT | 群主 uid (detail 表) | `u_xxx` |
| `60011` | INTEGER | 群等级/活跃度 | |

### 2.5 `recent_contact_v3_table` — 最近会话列表（49 行）
每行一个会话(消息列表里的一条)：对端号 / 最后消息 / 未读数 / 置顶 / 排序。

### 2.6 QQ 关联关系

```
group_msg_table.40033 (发送者QQ号) ──→ profile_info_v6.1002 ──→ 20002(昵称)
group_msg_table.40020 (发送者uid)  ──→ profile_info_v6.1000
group_msg_table.40021 (群号)       ──→ group_list/group_detail.60001 ──→ 60007(群名)
group_detail.60004 (群主QQ号)      ──→ profile_info_v6.1002
buddy_list.1002 ⊂ profile_info_v6.1002 (好友是联系人子集)
消息正文 40800(protobuf) ──→ 递归解 UTF-8 得文本; 富元素(图/@/引用)需按 element type 解
```

## 3. 抖音 / 头条 私信（ByteDance `com.ss.android.im` 框架，两者表结构**完全相同**）

> 抖音 `<uid>_im.db`(新版 SQLCipher 加密) / 头条 `<uid>_im.db`(明文)。同一套 IM 引擎。

### 3.1 `msg` — 私信消息

| 字段 | 类型 | 含义 | 说明 |
|---|---|---|---|
| `msg_uuid` | TEXT PK | 消息UUID | 本地唯一 |
| `msg_server_id` | BIGINT | 服务器消息ID | 全局唯一 |
| `conversation_id` | TEXT | **会话ID(关联键)** | 形如 `0:1:<uidA>:<uidB>`。**JOIN conversation_core/list.conversation_id** |
| `conversation_short_id` | BIGINT | 会话短ID | 数字版会话ID |
| `conversation_type` | INTEGER | 会话类型 | `0`=单聊 群聊另值 |
| `type` | INTEGER | 消息类型 | 文本/图片/卡片等 |
| `sender` | BIGINT | **发送者UID** | 抖音/头条用户uid(数字)。JOIN participant.sec_uid/uid |
| `sec_sender` | TEXT | 发送者 sec_uid | 加密版uid(字符串) |
| `content` | TEXT | **正文** | JSON(文本消息含 `{"text":"..."}`；其它类型是卡片JSON) |
| `created_time` | INTEGER | **时间戳(毫秒)** | |
| `read_status` | INTEGER | 已读状态 | |
| `index_in_conversation` | BIGINT | 会话内序号 | 排序用 |
| `deleted` | INTEGER | 是否删除 | |
| `ext` / `local_info` / `property_list` | TEXT | 扩展/本地态/属性 | JSON |

### 3.2 `conversation_core` — 会话/群核心信息

| 字段 | 含义 | 说明 |
|---|---|---|
| `conversation_id` | **会话ID(主键)** | = msg.conversation_id |
| `name` | 会话/群名 | 群聊才有 |
| `desc` / `notice` | 群简介/公告 | |
| `owner_id` / `creator_uid` | 群主/创建者 uid | |
| `create_time` | 创建时间 | |
| `mode` | 会话模式 | 单聊/群聊 |

### 3.3 `conversation_list` — 会话列表
`conversation_id` / `unread_count`(未读) / `last_msg_index`(最后消息) / `member_count`(成员数) / `updated_time` / `participant`(成员JSON) / `sort_order`(排序)。

### 3.4 `participant`（成员表，部分库有）
会话成员：`conversation_id` + 成员 `uid`/`sec_uid` + 群名片。是 msg.sender → 显示名的关联表。

### 3.5 抖音/头条 关联关系

```
msg.conversation_id ──→ conversation_core.conversation_id (取 name 会话/群名)
                    └─→ conversation_list.conversation_id (未读/排序/成员数)
msg.sender (uid) ──→ participant(同 conversation_id).uid ──→ 群名片/昵称
msg.content ──→ JSON 解析: 文本取 .text; 卡片取标题/链接
attchment.uuid ──→ msg(图片/文件消息的附件: local_uri/remote_uri/mime_type)
mention.uuid ──→ msg (这条消息@了谁: ids_str)
```

> **头条另有阅读类**：`news_article.db` 的 `article`(feed缓存) / `article_detail`(打开过的正文，标题在 `content` HTML 的 `<div class="tt-title">`) — 用户兴趣信号(点赞 `is_user_digg`/收藏 `is_user_repin`/阅读 `read_timestamp`)。

## 4. 四 App 横向对照（同一字段在哪）

| 语义 | 微信 | QQ | 抖音/头条 |
|---|---|---|---|
| 消息唯一ID | message.msgSvrId | 40001 | msg.msg_server_id |
| 正文 | message.content(文本/XML) | 40800(protobuf) | msg.content(JSON) |
| 发送者 | content前缀wxid / isSend | 40033(QQ号) | msg.sender(uid) |
| 收发方向 | isSend(0收/1发) | 40010 / 比对40033==自己 | 比对sender==自己 |
| 时间 | createTime(**毫秒**) | 40050(**秒**) | created_time(**毫秒**) |
| 会话/群 | talker(@chatroom) | 40021(群号) | conversation_id |
| 群名 | chatroom→(无名,需服务器) | group_info.60007 | conversation_core.name |
| 联系人昵称 | rcontact.nickname/conRemark | profile_info.20002 | participant 群名片 |
| 消息类型码 | type(1文本/3图…) | 40011(2普通/5,9系统) | type |

**入库映射**(→ PDH vault)：一条消息=1 event(occurredAt/actor=发送者person/participants/content.text)；
联系人=person(names=[昵称], identifiers.qq/wxid)；群=topic(name=群名)。详见 `scripts/android/pdh-*-ingest.mjs`。
