## 🚀 快速开始

1. **下载安装**

## 📋 系统概述

### ChainlessChain 是什么？

ChainlessChain 是一个去中心化的个人AI管理系统，集成了：

1. **知识管理** - 个人第二大脑，支持 Markdown、图片、PDF、Word 等多种格式
2. **AI助手** - 集成本地和云端LLM，支持RAG增强检索
3. **版本控制** - 基于Git的自动版本控制和多设备同步
4. **隐私保护** - U-Key硬件加密，端到端加密通信
5. **去中心化社交** - DID身份系统，P2P加密消息
6. **数字资产交易** - 知识付费、信用评分、智能合约

### 核心优势

- **数据主权**：数据完全属于你，存储在本地加密数据库
- **隐私安全**：U-Key硬件加密 + 端到端加密通信
- **离线优先**：所有核心功能离线可用
- **AI增强**：本地LLM，无需担心数据泄露
- **开放生态**：支持插件扩展，开源可审计

## 🔧 安装与配置

### 系统要求

**最低配置**：
- CPU: 4核心
- 内存: 8GB RAM
- 硬盘: 20GB 可用空间
- 系统: Windows 10/11, macOS 11+, Ubuntu 20.04+

**推荐配置**（用于本地LLM）：
- CPU: 8核心+
- 内存: 16GB+ RAM
- GPU: NVIDIA RTX 2060+ (6GB+ VRAM)
- 硬盘: 50GB+ SSD

### 安装步骤

#### Windows

1. 下载 `ChainlessChain-Setup-*.exe`
2. 双击运行安装程序
3. 选择安装路径（默认：`C:\Program Files\ChainlessChain`）
4. 完成安装，启动应用

#### macOS

1. 下载 `.dmg` 文件
2. 打开镜像，拖动到 Applications
3. 首次启动需要在"系统偏好设置 > 安全性与隐私"中允许
4. 启动应用

#### Linux

```bash
# 使用 AppImage
chmod +x ChainlessChain-*.AppImage
./ChainlessChain-*.AppImage

# 或使用包管理器（Ubuntu/Debian）
sudo dpkg -i chainlesschain_*.deb
chainlesschain
```

### 初始配置

#### 1. 设置主密码

首次启动时，系统会要求设置主密码：

- 密码长度至少8位
- 建议包含大小写字母、数字和特殊字符
- 此密码用于加密本地数据库
- **请务必牢记，遗忘后数据无法恢复**

#### 2. 配置数据存储

选择数据存储位置：

- **默认位置**：`~/ChainlessChain/data`
- **自定义位置**：选择其他路径（如外置硬盘、NAS）

#### 3. 配置LLM服务

选择AI模型服务：

**选项A：本地LLM（推荐，隐私优先）**
```
模型：Ollama
下载模型：qwen2:7b（约4GB）
GPU加速：自动检测
```

**选项B：云端LLM（性能优先）**
```
服务商：阿里云通义千问 / 智谱GLM / OpenAI等
API Key：输入你的API密钥
模型选择：qwen-max / glm-4 / gpt-4等
```

**选项C：混合模式**
- 日常对话使用本地模型
- 复杂任务使用云端模型

#### 4. U-Key 配置（可选）

如果你有U-Key硬件设备：

1. 插入U-Key到USB接口
2. 系统自动检测设备
3. 输入PIN码（默认：`123456`）
4. 完成设备绑定

**注意**：
- Windows：完整硬件支持
- macOS/Linux：仅模拟模式

1. 用户提问

3. 重排序（Rerank）选出最相关内容
4. 结合上下文生成回答

**配置选项**：
```yaml
检索设置:
  相似度阈值: 0.7 (0-1)
  返回结果数: 5
  重排序: 启用
  包含来源: 启用
```

**检索范围**：
- 全部笔记
- 指定文件夹
- 指定标签
- 指定时间范围

### 3. Git版本控制

ChainlessChain 内置 Git 版本控制，自动记录所有变更，支持多设备同步。

#### 3.1 自动版本控制

**默认行为**：
- 每次保存笔记自动提交（Auto-commit）
- 提交信息自动生成：`Update: 笔记标题`
- 支持关闭自动提交，手动控制

**查看历史**：
1. 右键点击笔记
2. 选择 "版本历史"
3. 浏览所有历史版本
4. 对比差异（Diff视图）
5. 恢复到任意历史版本

#### 3.2 Git同步

**配置远程仓库**：

1. 打开 "设置 > Git配置"
2. 选择仓库类型：
   - GitHub
   - GitLab
   - Gitee
   - 自建Git服务器
3. 输入仓库URL：`https://github.com/username/repo.git`
4. 认证方式：
   - HTTPS（用户名+密码/Token）
   - SSH（公钥认证）

**同步操作**：

- **推送（Push）**：将本地更改上传到远程
- **拉取（Pull）**：从远程下载更改
- **自动同步**：设置定时自动推拉（如每30分钟）

**冲突解决**：

当多设备同时修改同一笔记时：

1. 系统检测到冲突
2. 弹出冲突解决界面
3. 三栏对比视图：
   - 左侧：本地版本
   - 中间：合并结果
   - 右侧：远程版本
4. 手动选择保留内容
5. 确认解决，完成合并

#### 3.3 分支管理（高级）

```bash
创建分支: feature/new-idea
切换分支: main ↔ feature/new-idea
合并分支: feature/new-idea → main
删除分支: feature/new-idea
```

**应用场景**：
- 实验性内容：创建分支测试，满意后合并
- 多人协作：每人一个分支，定期合并
- 版本发布：维护release分支

### 5. U-Key安全

U-Key 是硬件安全设备，提供企业级加密和身份认证。

#### 5.1 U-Key 功能

**数据加密**：
- AES-256 硬件加密
- 密钥存储在U-Key中，不可导出
- 数据库自动加密

**身份认证**：
- 双因素认证（2FA）
- 硬件证书（X.509）
- DID身份绑定

**数字签名**：
- 笔记签名（防篡改）
- 交易签名（去中心化交易）
- 合约签名（智能合约）

#### 5.2 使用流程

**初始化U-Key**：
1. 插入U-Key
2. 系统检测设备
3. 设置PIN码（6-16位）
4. 生成密钥对
5. 完成初始化

**日常使用**：
1. 启动应用
2. 插入U-Key
3. 输入PIN码
4. 自动解密数据库
5. 正常使用

**备份密钥**（重要）：
1. 打开 "设置 > U-Key管理"
2. 点击 "导出恢复密钥"
3. 输入PIN码
4. 保存到安全位置（如纸质备份、密码管理器）
5. **切勿将恢复密钥存储在电脑上**

#### 5.3 安全最佳实践

1. **PIN码管理**：
   - 不使用生日、手机号等易猜测密码
   - 定期更换PIN码
   - 连续输错3次锁定设备

2. **设备保管**：
   - U-Key妥善保管，防止丢失
   - 不与他人共用U-Key
   - 丢失后立即远程撤销设备

3. **多设备备份**：
   - 准备备用U-Key
   - 导出恢复密钥
   - 定期测试恢复流程

#### 5.4 模拟模式（无硬件）

如果没有U-Key硬件，可以使用模拟模式：

1. 打开 "设置 > U-Key管理"
2. 选择 "模拟模式"
3. 设置模拟PIN码
4. 系统使用软件加密（安全性降低）

**注意**：模拟模式仅用于测试，生产环境请使用硬件U-Key。

### 6. P2P社交网络

ChainlessChain 支持去中心化社交，基于DID身份和P2P网络。

#### 6.1 创建DID身份

DID（Decentralized Identifier）是你的去中心化身份标识。

**创建流程**：
1. 打开 "社交 > 身份管理"
2. 点击 "创建DID"
3. 填写信息：
   - 昵称
   - 头像
   - 个人简介
4. 系统生成DID：`did:chain:abc123...`
5. 使用U-Key签名（可选）

**DID特性**：
- 全球唯一
- 自主控制
- 可验证
- 跨平台通用

#### 6.2 添加好友

**方式一：扫描二维码**
1. 面对面时，打开对方的DID二维码

### 9. MCP 集成

MCP (Model Context Protocol) 是扩展 AI 能力的开放协议，支持连接外部工具和数据源。

#### 9.1 启用 MCP

```
设置 > MCP 服务器 > 启用 MCP 系统
```

#### 9.2 支持的服务器

| 服务器 | 用途 | 安全级别 |
|--------|------|----------|
| **Filesystem** | 文件读写操作 | 中 |
| **PostgreSQL** | 数据库查询 | 高 |
| **SQLite** | 本地数据库访问 | 中 |
| **Git** | 仓库操作 | 中 |
| **Fetch** | HTTP 请求 | 中 |

#### 9.3 连接服务器

```
MCP 服务器 > 选择服务器 > 连接
```

**配置示例（Filesystem）**：
```json
{
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/data/notes"],
  "permissions": {
    "allowedPaths": ["notes/", "imports/"],
    "forbiddenPaths": ["private/", "keys/"],
    "readOnly": false
  }
}
```

#### 9.4 安全策略

- **白名单机制**：只有注册的服务器可以加载
- **路径限制**：禁止访问敏感目录（数据库、密钥等）
- **用户同意**：高风险操作需要确认
- **审计日志**：所有操作记录到日志

## ❓ 常见问题

### 1. 数据安全性如何保证？

- **本地加密**：SQLCipher AES-256加密数据库
- **传输加密**：TLS 1.3 + 端到端加密
- **硬件保护**：U-Key硬件加密（可选）
- **开源审计**：代码开源，可自行审计

### 2. 可以多人协作吗？

可以，通过以下方式：
- **Git同步**：共享Git仓库
- **项目分享**：生成分享链接
- **P2P协作**：添加好友，实时协作（开发中）

### 3. 支持哪些LLM模型？

**本地模型**（Ollama）：
- Qwen2 (7B/14B/72B)
- Llama3 (8B/70B)
- Mistral
- 等100+模型

**云端模型**：
- OpenAI (GPT-4, GPT-3.5)
- 阿里云通义千问
- 智谱GLM
- 百度千帆
- 等14+服务商

### 4. 数据存储在哪里？

- **默认位置**：`~/ChainlessChain/data`
- **可自定义**：设置中更改路径
- **完全本地**：不强制上传云端

### 5. 如何迁移数据？

**方式一：备份恢复**
1. 旧设备：设置 > 备份 > 导出
2. 新设备：设置 > 备份 > 导入

**方式二：Git同步**
1. 旧设备：配置Git远程仓库，推送
2. 新设备：配置相同仓库，拉取

**方式三：直接复制**
- 复制 `data` 文件夹到新设备

### 6. 支持移动端吗？

- **当前版本**：仅桌面端（Windows/macOS/Linux）
- **规划中**：移动端开发进度10%，预计v1.0发布

### 7. 是否收费？

- **个人版**：完全免费
- **企业版**：高级功能（如团队管理、审计日志）需付费
- **云服务**：可选购买云端LLM额度

### 8. 如何卸载？

**Windows**：
- 控制面板 > 程序和功能 > 卸载ChainlessChain
- 数据不会自动删除，手动删除 `~/ChainlessChain`

**macOS**：
- 拖动应用到废纸篓
- 手动删除 `~/ChainlessChain`

**Linux**：
```bash
sudo apt remove chainlesschain
rm -rf ~/ChainlessChain
```

## 🔧 故障排除

### 问题1：应用无法启动

**可能原因**：
- 端口被占用（5173）
- 数据库损坏
- 权限不足

**解决方法**：
1. 检查端口占用：`netstat -ano | grep 5173`
2. 重置数据库：删除 `data/chainlesschain.db`（会丢失数据）
3. 以管理员身份运行

### 问题2：LLM无法连接

**本地Ollama无法连接**：
1. 检查Ollama服务：`curl http://localhost:11434`
2. 启动Ollama：`ollama serve`
3. 拉取模型：`ollama pull qwen2:7b`

**云端API无法连接**：
1. 检查API Key是否正确
2. 检查网络连接
3. 查看控制台错误日志

### 问题3：同步失败

**Git推送失败**：
1. 检查网络连接
2. 检查仓库URL和认证信息
3. 查看Git日志：设置 > Git配置 > 查看日志

**冲突无法解决**：
1. 备份本地数据
2. 重置到远程版本
3. 手动合并内容

### 问题4：U-Key无法识别

**Windows**：
1. 检查驱动：设备管理器查看USB设备
2. 重新安装驱动：运行 `SIMKeySDK/driver-install.exe`
3. 换个USB接口

**macOS/Linux**：
- 当前仅支持模拟模式
- 硬件支持开发中

### 问题5：性能问题

**应用卡顿**：
1. 关闭不需要的功能（如自动同步）
2. 清理缓存：设置 > 高级 > 清理缓存
3. 减少开启的笔记数量

**LLM响应慢**：
1. 本地模型：检查GPU是否启用
2. 云端模型：切换到更快的服务商
3. 减少RAG检索数量

### 问题6：数据丢失

**紧急恢复**：
1. 查看自动备份：设置 > 备份
2. 从Git历史恢复：版本历史 > 恢复
3. 使用恢复密钥（如果有U-Key备份）

**预防措施**：
- 启用自动备份
- 配置Git远程仓库
- 定期导出重要笔记
