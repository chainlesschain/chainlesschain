### 1️⃣ FCM 真接入（最容易，~30 分钟）

**目的**：让桌面端发的 push 通知能投递到 Android（M3 D3 现已有本地 channel + 骨架 + 35 测，缺 google-services.json 凭证）。

**完整说明**：[`android-app/docs/M3_FCM_SETUP.md`](../android-app/docs/M3_FCM_SETUP.md)（5 步详细 + 完整 code snippets）。

**简化操作流**：

- [ ] **步骤 1：Firebase Console 建项目**
  - 打开 https://console.firebase.google.com
  - 新建项目（命名 `chainlesschain` 或同名）
  - 添加 Android app，包名填 `com.chainlesschain.android`
  - 取 debug keystore 的 SHA-1：
    ```bash
    keytool -list -v -alias androiddebugkey -keystore ~/.android/debug.keystore -storepass android -keypass android | grep SHA1
    ```
  - 下载 `google-services.json`

- [ ] **步骤 2：放置凭证**
  - `cp google-services.json android-app/app/`（or `app/src/debug/`）
  - `build.gradle.kts` line 26-30 已配置 conditional 检测，会自动启用 Firebase plugins（gradle sync 会打印 `✓ Firebase enabled`）

- [ ] **步骤 3：加 firebase-messaging dep**
  - 编辑 `android-app/app/build.gradle.kts` line ~394-400 的 conditional Firebase block，加一行：
    ```kotlin
    if (hasGoogleServices) {
        implementation(platform("com.google.firebase:firebase-bom:32.7.0"))
        implementation("com.google.firebase:firebase-crashlytics-ktx")
        implementation("com.google.firebase:firebase-analytics-ktx")
        implementation("com.google.firebase:firebase-messaging-ktx")  // ← 加这行
    }
    ```

- [ ] **步骤 4：写 CcFirebaseMessagingService.kt**
  - 文件：`android-app/app/src/main/java/com/chainlesschain/android/push/CcFirebaseMessagingService.kt`
    ```kotlin
    package com.chainlesschain.android.push

    import com.google.firebase.messaging.FirebaseMessagingService
    import com.google.firebase.messaging.RemoteMessage
    import dagger.hilt.android.AndroidEntryPoint
    import javax.inject.Inject

    @AndroidEntryPoint
    class CcFirebaseMessagingService : FirebaseMessagingService() {
        @Inject lateinit var service: CcPushNotificationService

        override fun onMessageReceived(msg: RemoteMessage) {
            service.onRemoteData(msg.data)
        }

        override fun onNewToken(token: String) {
            service.onTokenChanged(token)
        }
    }
    ```

- [ ] **步骤 5：Manifest 注册**
  - 编辑 `android-app/app/src/main/AndroidManifest.xml`，在 `<application>` 内加：
    ```xml
    <service
        android:name=".push.CcFirebaseMessagingService"
        android:exported="false"
        android:directBootAware="false">
        <intent-filter>
            <action android:name="com.google.firebase.MESSAGING_EVENT"/>
        </intent-filter>
    </service>
    ```

- [ ] **验证**：`./gradlew :app:assembleDebug` 通过 + 装到设备
- [ ] **送达测试**：Firebase Console → Cloud Messaging → "Send your first message"：
  - Target：Android app
  - Notification: **留空**（必须 data-only，不能用 notification 字段）
  - Additional options → Custom data：
    ```
    type:        cowork.request
    taskId:      test-task-001
    summary:     测试通知
    agentName:   test
    ```
  - 期望：状态栏弹"Cowork 任务等待审批" 通知（蓝色 PRIMARY container）

**风险点**：
- 国内可达性（设计 §9.1）— FCM 在墙内不稳定，v1.1 计划接 OPPO/小米/华为统一推送
- VPN / 网络代理可能干扰 token 注册

### 2️⃣ M3 真机 E2E（需 Android 真机 + 桌面联机；~1h）

**前置**：FCM 可选；其他 4 件不需要 FCM 凭证。

**配对**：
- [ ] 桌面启动 `desktop-app-vue/`：`npm run dev`
- [ ] Android 设备扫描桌面 QR 码完成 P2P pairing
- [ ] 验证 `SyncCoordinator` 已启动（Android Logcat: `[SyncCoordinator] peer connected`）

**5 件 L2 验证矩阵**：

- [ ] **VoiceMode（M3 D1）**
  - 进 Settings → LLM → 配置火山 SeedASR API Key + resourceId
  - 进入 VoiceModeScreen（v1.0 demo screen；future 集成 ChatPanel）
  - 长按麦克风说一句中文（5~15s）→ 抬起
  - 期望：状态卡显示 "正在识别…" → "你说: <text>" → "助手 思考中…" → "🔊 正在朗读…" → "你说+助手" 双气泡
  - 打开 continuous 开关再试：期望 TTS 完后自动重启录音

- [ ] **CameraOCR（M3 D1）**
  - 进入 CameraOcrScreen
  - 点"拍照" → 系统相机拍发票/名片 → 回到屏
  - 期望：状态卡走 Encoding → Recognizing → Recognized
  - 编辑标题 + 内容 → "保存到 KB"
  - 期望：状态卡 → Saving → Saved；桌面 KB 端看到新 note (`shared, ocr, mobile` tags)
  - 打开附带位置开关再拍一张 → 验证 note 头部有 `📍 lat,lon ±Xm @ time` blockquote

- [ ] **LocationTagger（M3 D2）**
  - 同上 CameraOCR 附带位置流程（覆盖 LocationTagger.fetchOnce 调用）
  - 或独立验证：从 Settings 启动 `LocationForegroundService`（`context.startForegroundService(LocationForegroundService.start(context))`）
  - 验证：状态栏出现"位置采集中"通知 + 锁屏期间 5 分钟仍持续

- [ ] **ShareReceiver（M3 D2）**
  - 任意 app（浏览器/微信/邮件）分享一条文字 → 选择 ChainlessChain
  - 期望：Activity 闪一下立刻 finish（NoDisplay theme）
  - 等 30s（SyncCoordinator 周期）→ 桌面 KB 出现新 note（`shared, text` tags，标题 = 分享 subject 或 "Shared text"）
  - 试分享 URL / 图片 / PDF 各一次，验证 5 种 SharePayload 类型对应的 note 格式

- [ ] **PushNotifier（M3 D3，需 1️⃣ FCM 完成）**
  - 从 Firebase Console 发 4 种 type 各一条（见 1️⃣ 步骤 5 协议）
  - 期望：4 类通知各自正确 channel + icon + 点击回 MainActivity（v1.0 路由器未实现 deep-link 拆分，预期回主屏）

**通过标准**：5 件全 ✅，每件 30 秒内能复现（不算 LLM API 调用延迟）。

### 3️⃣ M4 D2 真机 E2E（需 Android 真机 + 桌面联机；~1.5h）

**前置**：2️⃣ 配对完成。

**完整说明**：[`docs/design/Android_M4_D2_E2E.md`](design/Android_M4_D2_E2E.md)（设计文档原文：7 demo scenarios）。

**4 个"最后一公里"环节**：

- [ ] **真 WebRTC DataChannel 反向 RPC**
  - 桌面调 `command-router` 触发高风险 method（如 `marketplace.purchase`）
  - 期望：Android Logcat: `approval.request routed: id=apr-... category=Marketplace`

- [ ] **真 Compose ApprovalDialog 4 类 category**
  - 触发以下 method 各一次，期望对应 dialog 渲染：
    - `sign.request` → "签名请求确认" + Security icon + primary 色
    - `cowork.spawnTeam` → "Cowork 任务审批" + Groups icon + primary 色
    - `marketplace.purchase` → "交易审批" + ShoppingCart icon + tertiary 色
    - `system.shutdown` → "关键操作审批" + Warning icon + error 色

- [ ] **真 BiometricPrompt**
  - 高风险 method 默认 requireBiometric=true → 点"同意并验证" → 系统指纹/Face Unlock 弹出
  - 验证成功 → 桌面收到 `approved:true`
  - 失败/取消 → 桌面收到 `approved:false, deniedReason:"biometric-failed: ..."`

- [ ] **真 StrongBox 签名**
  - 设备需支持 StrongBox（Pixel 3+ / Samsung Galaxy S10+ / 部分小米/华为旗舰）
  - 在 ApprovalDialog 验证 tier chip 显示 "StrongBox" 而非 "TEE" / "Software"
  - `cc-mobile` Settings → DID Wallet 显示 KeyTier.StrongBox

- [ ] **ProgressViewer 验证**
  - 桌面触发一个 Cowork 任务（如 spawnTeam 后跟踪 task progress）
  - 期望：桌面通过 `task.update` reverse-RPC 推快照 → Android Logcat: `task.update: id=... state=Running progress=...`
  - 进入 ProgressViewerScreen → 见 task 卡片 + LinearProgressIndicator + 状态 chip
  - 任务完成 → 卡片显示"已完成 · HH:mm:ss" + Close 按钮
  - 失败 → 卡片显示 errorMessage + 红色 chip

**通过标准**：所有 4 环节 + ProgressViewer 各自能正常运行；桌面 → Android → 桌面 round-trip < 2s（同 LAN）。

### 4️⃣ M6 性能实测回填（需 Android 真机 + 测试工具；~1h）

**前置**：装好 Android Studio + 真机 + `adb` 可用。

**完整说明**：[`docs/design/Android_M6_Performance_Validation.md`](design/Android_M6_Performance_Validation.md)（设计文档：完整方法学）。

**8 项预算回填**：

| 指标 | 目标 | 工具 | 命令/路径 |
|---|---|---|---|
| 冷启动 → DID 解锁 | <1500ms | Macrobenchmark | `./gradlew :benchmark:connectedCheck`（待加 benchmark module）或手动 `adb shell am start -W` |
| 暖启动 | <400ms | 同上 | 同上 |
| 24h 后台耗电 | <3% | Battery Historian | 装 `https://github.com/google/battery-historian`，跑 `adb bugreport` 24h 后分析 |
| 包体大小 | <30MB | Gradle | `./gradlew :app:assembleRelease && ls -lh app/build/outputs/apk/release/*.apk` |
| 同 LAN REMOTE p50 | <200ms | 内嵌埋点 | Android Logcat + 桌面 logger 时间戳 diff |
| 跨 NAT REMOTE p50 | <800ms | 内嵌埋点 + TC 弱网 | `tc qdisc add dev wlan0 root netem delay 200ms loss 5%` |
| StrongBox 签名 | <30ms | StrongBoxKeyManager 测试模式 | 已有 microbenchmark API（M2 D1） |
| ASR 首字延迟 | <500ms | VoiceMode pipeline 埋点 | Logcat: `VolcAsr.submit reqId=... → poll reqId=... status=20000000` |

- [ ] 把 8 项实测值填回 `Android_M6_Performance_Validation.md` 末尾的"实测结果"表
- [ ] 偏差 >20% 的指标加注解说明（如 "ASR 首字延迟 850ms — 火山 SeedASR 网络延迟主导，<500ms 需切本地 Whisper（v1.1）"）

**通过标准**：8 项中 ≥6 项达标，剩下偏差有文档解释。

## 中止条件

如 1️⃣~4️⃣ 任一发现 **架构性缺陷**（如 ApprovalCommandRouter 拿不到 method 字段、ProgressViewer state 不更新），暂停 GA：

1. 抓 Logcat + 桌面 logger 完整 round-trip
2. 复现到 minimum repro
3. 看是 8 commits 中的哪一个引入 → 在该 commit branch 修
4. 重跑相关测试套件验证修复
5. cherry-pick / amend 后再回 GA 流程

---

**最后一句**：v1.0 RFC 落地 7 commits 是把 "对桌面 skill 数量的弱化追赶" 重新定位为 "DID 钱包 + 移动捕获 + REMOTE 遥控器" 三层模型的 code 部分；真机验收是把这套模型在硬件上跑起来的最后一公里。**完成 5 项即 v1.0.0 GA**。
