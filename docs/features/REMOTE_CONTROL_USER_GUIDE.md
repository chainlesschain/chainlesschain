## 简介

### 什么是远程控制系统？

ChainlessChain 远程控制系统允许您通过 Android 手机远程控制您的 PC 电脑，执行以下操作：

- 🤖 **AI 对话**: 在手机上与 PC 端的 AI 助手对话
- 🔍 **RAG 搜索**: 在手机上搜索 PC 端的知识库
- 🎮 **Agent 控制**: 在手机上控制 PC 端的 AI Agent
- 📸 **远程截图**: 获取 PC 屏幕截图
- 📊 **系统监控**: 查看 PC 的 CPU、内存、磁盘使用情况
- 📝 **命令历史**: 查看和重放历史命令

### 核心特性

✅ **完全去中心化**: 基于 P2P 网络，无需中心服务器
✅ **端到端加密**: 使用 Signal 协议保护通信安全
✅ **DID 身份认证**: 基于去中心化身份系统
✅ **离线队列**: 支持离线发送命令，PC 上线后自动执行
✅ **完整日志**: PC 端记录所有命令日志和统计数据

## 快速开始

### 前置条件

#### PC 端要求
- ✅ Windows 10/11, macOS 10.15+, 或 Linux
- ✅ ChainlessChain 桌面应用 v0.27.0+
- ✅ 网络连接（用于 P2P 通信）

#### Android 端要求
- ✅ Android 12 或更高版本
- ✅ ChainlessChain Android 应用 v0.27.0+
- ✅ 网络连接（Wi-Fi 或移动数据）

### 首次设置

#### 步骤 1: 启动 PC 端应用

1. 打开 ChainlessChain 桌面应用
2. 确保 P2P 网络已启动（状态栏显示"已连接"）
3. 记下您的 DID（设置 → 身份信息）

#### 步骤 2: 配置 Android 端

1. 打开 ChainlessChain Android 应用
2. 进入 "远程控制" 页面
3. 点击 "添加设备"
4. 输入 PC 端的 DID 或扫描二维码
5. 等待连接建立（绿色指示灯）

#### 步骤 3: 测试连接

1. 在 Android 端点击 "系统状态"
2. 如果能看到 PC 的 CPU、内存信息，说明连接成功！

## 常见功能说明

### 5.1 离线命令队列

**问题**: 如果 PC 端离线，手机能发送命令吗？

**答案**: 可以！命令会进入离线队列，等 PC 上线后自动执行。

**使用方法**:
1. PC 离线时，正常在手机上发送命令
2. 命令状态显示为 "排队中 ⏳"
3. PC 上线后，命令自动执行
4. 手机收到执行结果通知

**注意事项**:
- 离线队列最多保存 1000 条命令
- 超过 7 天的命令会自动清理
- 建议重要命令在 PC 在线时执行

### 5.2 多设备支持

**问题**: 可以用多个手机控制同一台 PC 吗？

**答案**: 可以！一台 PC 支持多个 Android 设备连接。

**使用方法**:
1. 每个 Android 设备独立添加 PC 的 DID
2. 每个设备有独立的命令历史
3. PC 端会记录所有设备的命令日志

### 5.3 权限管理

**问题**: 如何控制哪些命令可以执行？

**答案**: PC 端有完善的权限控制系统。

**权限级别**:
- **公开命令**: 所有已连接设备都可执行（如 chat, ragSearch）
- **受限命令**: 需要特定权限（如 screenshot, getStatus）
- **禁止命令**: 危险命令被禁用（如 execCommand）

**配置方法**:
1. PC 端打开 "设置" → "远程控制权限"
2. 为每个设备配置权限级别
3. 启用/禁用特定命令

## 常见问题

### Q1: 连接一直失败怎么办？

**可能原因**:
1. PC 端应用未启动或 P2P 网络未运行
2. DID 输入错误
3. 网络防火墙阻止连接
4. PC 和手机不在同一网络

**解决方法**:
1. 确保 PC 端应用正在运行，状态栏显示 "已连接"
2. 重新扫描 PC 端的 DID 二维码
3. 检查防火墙设置，允许应用访问网络
4. 尝试使用移动数据（如果 Wi-Fi 有限制）

### Q2: AI 对话响应很慢？

**可能原因**:
1. PC 端 AI 模型正在加载
2. 网络延迟较高
3. PC 端性能不足

**解决方法**:
1. 首次使用模型时需要加载，请等待 1-2 分钟
2. 检查网络连接质量
3. 切换到更小的模型（如 qwen2:7b）
4. 关闭 PC 上的其他占用资源的程序

### Q3: 截图功能不可用？

**可能原因**:
1. PC 端截图权限未授予
2. 网络带宽不足
3. PC 端使用的是 Linux 系统（部分环境不支持）

**解决方法**:
1. macOS: 系统偏好设置 → 安全性与隐私 → 屏幕录制，勾选 ChainlessChain
2. 使用 Wi-Fi 连接，避免使用移动数据
3. Linux 用户: 确保安装了 `scrot` 或 `gnome-screenshot`

### Q4: 命令历史记录丢失？

**可能原因**:
1. 手机端数据库损坏
2. 应用被系统清理
3. 手动清除了应用数据

**解决方法**:
1. 命令历史仅存储在手机本地，无法恢复
2. PC 端的日志完整保留，可以导出查看
3. 建议定期导出重要命令历史

### Q5: 如何更换控制的 PC？

**步骤**:
1. 在 Android 端进入 "远程控制" 页面
2. 点击 "设备管理"
3. 点击 "移除当前设备"
4. 重新添加新的 PC DID

### Q6: 命令执行失败，显示 "权限不足"？

**原因**: PC 端禁用了该命令或您的设备没有权限。

**解决方法**:
1. 联系 PC 管理员开启权限
2. PC 端: 设置 → 远程控制权限 → 允许该命令

## 故障排查

### 7.1 连接问题

#### 症状: "连接中..." 一直不成功

**排查步骤**:

1. **检查 PC 端状态**
   ```
   - 打开桌面应用
   - 查看状态栏: 应显示 "P2P: 已连接"
   - 如果显示 "离线" 或 "连接中"，重启应用
   ```

2. **检查网络连接**
   ```
   - PC 和手机都有网络连接
   - 尝试 ping PC 的 IP 地址（如果在同一局域网）
   - 尝试切换网络（Wi-Fi ↔ 移动数据）
   ```

3. **检查 DID 配置**
   ```
   - 重新扫描 PC 端的 DID 二维码
   - 确保输入的 DID 完整无误
   ```

4. **重启应用**
   ```
   - 完全关闭 Android 应用（清除后台）
   - 重新打开应用
   - 尝试重新连接
   ```

#### 症状: 连接成功但命令无响应

**排查步骤**:

1. **检查 PC 端日志**
   ```
   - 打开 "命令日志" 页面
   - 查看是否收到命令
   - 检查错误信息
   ```

2. **检查命令权限**
   ```
   - 设置 → 远程控制权限
   - 确认该命令已启用
   ```

3. **检查 AI 服务**（仅限 AI 命令）
   ```
   - 确保 Ollama 服务正在运行
   - 确保模型已下载
   ```

### 7.2 性能问题

#### 症状: 命令执行很慢（超过 10 秒）

**排查步骤**:

1. **检查网络延迟**
   ```
   - 使用 "系统状态" 命令测试响应时间
   - 如果延迟 > 2 秒，检查网络质量
   ```

2. **检查 PC 端性能**
   ```
   - 查看 PC 端 CPU/内存使用率
   - 关闭其他占用资源的程序
   ```

3. **切换到更快的模型**（AI 命令）
   ```
   - 使用更小的模型（如 qwen2:7b）
   - 避免使用大模型（如 llama3:70b）
   ```

### 7.3 数据问题

#### 症状: 命令历史显示不完整

**排查步骤**:

1. **检查数据库**
   ```
   - Android: 设置 → 应用 → ChainlessChain → 存储
   - 查看数据大小（正常应 > 1 MB）
   ```

2. **检查过滤器**
   ```
   - 历史页面顶部的过滤器
   - 重置所有过滤条件
   ```

3. **清理缓存**
   ```
   - 设置 → 应用 → ChainlessChain → 清除缓存
   - 重新打开应用
   ```
